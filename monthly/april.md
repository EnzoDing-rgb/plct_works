## 3. 下一步与准备

- 准备进行下一个更大的项目，两个项目作为保险。
- 购买了火山引擎和 Cursor 的大模型服务。
- 研究了本地高性能服务器环境运行开源大模型，降低开发成本，同时也是 LLM 推理服务的双保险。
  - 服务器与环境：A800-SXM4-80GB 单卡；CPU 2×8358（128 线程）；内存 1TB；NFS 存储 118TB；Docker 容器直通物理 GPU。现象“显存 94% 占用但 GPU 利用率 0%”＝模型已加载占坑但暂无推理请求。
  - 现状与瓶颈：Qwen3.5-27B BF16 权重约 ~52GB，导致 KV Cache 池偏小，并发/长上下文被显存硬卡（核心约束不是 `max-model-len`，而是 KV 池总预算主要由 `--gpu-memory-utilization` + 权重大小决定）。
  - 关键结论（ROI 排序）：`--kv-cache-dtype fp8`（几乎最划算，常见可近似并发翻倍）→ `--enable-prefix-caching`/`--enable-chunked-prefill`（场景性收益）→ 切换 FP8 权重模型（长期收益最大）→ Turbo GRT（KV 3–4bit 压缩，长上下文/高并发收益最大，需 vLLM 0.20+）→ DFlash/DDTree（更偏压测/实验，不作为生产默认）。
  - 为什么量化能提速：通常是 decode 阶段“显存带宽/搬数据”瓶颈下降（不是算力变强）；对数据中心卡收益更稳但不一定像消费卡那样夸张。
  - 生产默认选型倾向：稳定优先仍是 **FP8 + MTP**（多并发友好、工具调用更稳）；DFlash 在并发/工具调用/维护成本上风险更大，建议只在隔离压测环境验证。
  - 升级/验证策略：先 65K 稳跑 → 131K → 262K，每次只动一个旋钮（长度/并发/显存利用率）；用自建回放集对比 tokens/s、TTFT、P95/P99、工具调用成功率与一次通过率，质量下降超过阈值（如 2%–5%）就回退；保留停服/回滚命令确保可恢复。

## 观众要做什么

下面按方案编号，用**第二人称**写清楚：你站在这块板子前，**具体要动什么、看什么**。不用懂背后的技术名词。

| 方案 | 你要做什么 |
| --- | --- |
| **01 画画** | 在**左边**随手画几笔；看**右边**变成什么风格。**有鼠标或触摸屏就用手画**；只有键盘就按现场说明用光标挪着画。可以顺便看界面上写的「算了多久」「跟不跟手」。 |
| **02 推理桌** | 翻开线索卡、采纳证据，看五盏煤气灯的“嫌疑值”怎么变化；你还可以按 **U** 撤回证据，体验推理是可回滚、可验证的。 |
| **03 文风万花筒** | 用键盘输入一句大白话，回车后看它在九宫格里同时变成九种文风。按 **1–9** 放大某一格，按 **R** 在同一文风下再来一版。 |
| **04 天价废话** | 输入你身上最不值钱的东西，回车后看它被“包装成奢侈品档案卡”。按 **1/2/3** 让它更贵、换故事、出联名款，越按越离谱。 |
| **05 真假鉴定** | 先选主题（皮具/腕表/香水/珠宝），再在左右两张档案卡里猜 A/B 哪个是真的；看揭晓时 AI 边打字边解释你为什么会被骗。 |
